# Advanced OkLCH Palette Generator — Постмортем и Документация Проекта

Этот документ описывает историю создания, алгоритм работы и технические детали двух версий генератора палитр, созданных для Gravity UI (с уклоном в современные веб-методы). Документ предназначен для онбординга или возобновления работы над проектом.

## Исторический Контекст
1. **Первоначальный запрос** был направлен на изучение [Branding Palette Generator из открытого исходного кода Gravity UI](https://github.com/gravity-ui/uikit) (`src/stories/Branding/PaletteGenerator/`). 
2. Оригинальный алгоритм Gravity UI базировался на смешивании (mix) основного `Color` с контрастными цветами фона `Low/High Contrast Base` через матрицу прозрачностей `(a)` и контрастности `(c)` (шаги от 50 до 1000). Пространство смешивания — **RGB** (с помощью библиотеки `chroma-js`). Ограничение подхода Gravity: в RGB промежуточные цвета между ярким цветом и белым/черным часто становятся тусклыми (muddy colors) из-за прямолинейной математики.
3. Был запрошен персональный инструмент-генератор на основе алгоритма Gravity, но с современным UI. Он был реализован в `index.html`.
4. В генератор добавлены функции **OkLCH**. Обычное пространство RGB не является перцептивно равномерным. Переход к пространству OkLCH гарантировал одинаковый визуальный вес всех оттенков. Появились 3 формата вывода (`HEX`, `OkLCH`, `CHL`).
5. Заменен вывод CSS (SASS mixin) на прямое **генерирование вектора (SVG Экспорт)** для прямой и удобной вставки в Figma.
6. Вскрылась проблема линейного интерполирования `chroma.mix` в пространстве `oklch` (как в Gravity UI, так и в нашем порту): интерполяция `Hue` по окружности цветового цилиндра приводила к сдвигам оттенка (Hue), создавая мертвые зоны на промежуточных светлых тонах.
7. Создан `index2.html` с реализацией **продвинутого "Hue Shift" алгоритма** и динамического контекстного фона (Background Sync). Вдохновлено подходом дизайн-систем Tailwind CSS и Radix UI.

## Архитектура файлов проекта
Проект состоит из двух автономных HTML-файлов (оба работают без сборщика, используют `chroma.js` через CDN):

*   **`index.html`** (Оригинальная OkLCH версия): 
    *   Смешивает цвета нативно через векторное пространство OkLCH: `chroma.mix(color, base, ratio, 'oklch')`. 
    *   Фон правой панели статичен (темно-серый `#1a1c23`).
    *   Служит эталоном линейного перцептивного смешивания. Могут присутствовать паразитные смещения оттенка из-за интерполяции оси `h` (Hue).

*   **`index2.html`** (Advanced версия с Hue Shift):
    *   Динамический цвет фона у панели предпросмотра (`previewContent`), соответствующий полю `Low Contrast Base (Фон)`.
    *   Интеллектуальный контроль за `Hue` (сохраняет базовый оттенок и слегка закручивает его на концах спектра).
    *   Главная точка интереса — функция `interpolateOklchShifted(baseColorHex, targetBgHex, ratio)`.

## Ключевые компоненты и Константы

### Матрица Весов (Gravity UI Core)
Ядро обоих генераторов опирается на константную матрицу от 50 до 1000.
*   `a` (Alpha): Вес базового цвета (1.0 — абсолютный базовый цвет `Main Color`).
*   `c` (Contrast Direction): Направление смешивания. `c: -1` (светлые тона, смешивание с `Low Contrast Base`), `c: 1` (темные тона, смешивание с `High Contrast Base`). Шаг 550 является пиковым `a: 1, c: 1`.

### Алгоритм `interpolateOklchShifted` (index2.html)
Это сердце продвинутого генератора, решающее проблему грязных оттенков (Muddy Colors).
```javascript
// Логика функции:
1. Разбиваем BaseColor (основной цвет) и BgColor (фон) на их составляющие [L, C, H] в OkLCH.
2. Интерполируем (смешиваем) только каналы L (Светлота) и C (Насыщенность/Хрома) по переданному ratio.
3. Игнорируем Hue (оттенок) фонового цвета полностью, он не смешивается математически. За основу берется исключительно Hue основного цвета.
4. Добавляем "Температурный сдвиг" (Hue Shift):
   - Если смешивание идет со светлым фоном (белым), Hue "базового цвета" искусственно закручивается (например: теплый сдвиг вверх) пропорционально расстоянию от базового цвета.
   - Если фон темный (ближе к черному), Hue сдвигается вниз (холодные тени).
   - Максимальный сдвиг установлен на 15 градусов на дальних концах диапазона. 
```

### SVG Exporter (Экспорт в Figma)
Логика генерации SVG собирает строку `<svg>` на лету, перебирая сгенерированные данные `currentPaletteData`.
*   Рассчитывает общую ширину/высоту холста на основе фиксированных отступов (padding = 20) и размеров плашек (`200x80`).
*   Формирует `<rect>` и `<text>` теги для каждого узла.
*   Конвертирует строку в `Blob` (тип `image/svg+xml`) и скачивает средствами `URL.createObjectURL(blob)`.

## Как продолжить работу над проектом

### Развитие UI / Логики
1. **Настройка кривой Hue Shift**: Сейчас он установлен на ±15 градусов жестко. Можно вынести настройки `shiftAmount` в Sidebar как ползунки "Temperature Shift (Light/Dark)".
2. **Экспорт CSS**: Вернуть экспорт CSS переменных параллельно c кнопкой SVG для разработчиков (Tailwind theme / CSS root vars).
3. **Хранилище (Local Storage)**: Добавить сохранение последнего введенного Hex-кода в `localStorage`, чтобы при перезагрузке страницы введенные цвета не сбрасывались в синий по умолчанию.

### Дальнейшее погружение Oklch / Tailwind
Если вам потребуется более тонкий контроль (как в `colorgaffle` или Radix Colors), следующим шагом будет отказ от линейных массивов `colorsMap` и переход на Безье-кривые для расчета шагов `L` (Lightness), чтобы иметь не 20 шагов, а возможность генерировать бесконечно гладкие градиенты.
