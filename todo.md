## 🎨 Визуальная компенсация яркости (Perceptual Weight Matching)

При генерации нейтральных цветов (Grays) на основе насыщенных акцентов, использование идентичного параметра $L$ (Lightness) из OKLCH недостаточно. Из-за **эффекта Гельмгольца-Кольрауша** насыщенные цвета кажутся человеческому глазу более яркими и «энергичными», чем серые той же физической светлоты.

### Проблема «Легкого Серого»
Если создать серый цвет с тем же $L$, что и у синего акцента, серая кнопка будет выглядеть визуально «легче» и слабее. Для достижения **равного визуального веса** нейтральный цвет должен быть темнее насыщенного.



### Таблица коэффициентов коррекции ($K_{weight}$)

Алгоритм должен корректировать целевой $L_{gray}$ в зависимости от цветового тона ($h$) исходного цвета:

| Hue ($h$) | Пример цвета | Коэффициент $K$ | Обоснование |
| :--- | :--- | :--- | :--- |
| **240–280** | Синий / Индиго | **0.88 – 0.90** | Самая высокая визуальная энергия. Требует сильного затемнения серого (на ~11%). |
| **0–30** | Красный / Оранжевый | **0.92 – 0.94** | Средняя коррекция. Красный тяжелее желтого, но легче синего. |
| **90–140** | Зеленый / Лайм | **0.96 – 0.98** | Минимальная коррекция. Глаз максимально чувствителен к зеленому спектру. |
| **60–90** | Желтый | **1.00** | Желтый — самый светлый. Серый аналог может быть идентичен по $L$. |



### Алгоритм «Grey Balance» (Logic for AI)

При написании программного кода для генерации цветов, AI должен использовать следующую логику компенсации:

```python
# Рассчитываем визуально сбалансированный серый
# target_L = исходный Lightness акцентного цвета
# target_C = насыщенность акцентного цвета (от 0 до 0.4 в OKLCH)

def get_balanced_gray_L(target_L, target_C, hue):
    # Определяем чувствительность в зависимости от тона (Hue)
    # Для синих оттенков штраф выше
    if 240 <= hue <= 280:
        sensitivity = 0.28  # Высокая компенсация для синего
    elif 0 <= hue <= 40:
        sensitivity = 0.15  # Средняя для красного
    else:
        sensitivity = 0.05  # Низкая для желто-зеленого
    
    # Итоговая формула коррекции
    balanced_L = target_L - (target_C * sensitivity)
    return max(0, min(1, balanced_L))
